# Task Workflow System - Quick Start

복잡한 구현 작업의 생명주기를 관리하고 세션 간 문맥을 유지하는 시스템입니다.

## 🚀 5초 요약

```
#구현 작업명             → 작업 시작 (문서 자동 생성)
#구현재개 작업명         → 중단된 작업 이어하기
#구현완료 작업명         → 작업 완료 및 아카이빙
#구현폐기 작업명 사유    → 작업 폐기
#구현목록               → 진행 중인 작업 목록
```

## 📖 기본 사용법

### 1. 작업 시작
```
User: "#구현 유저랭킹기능 Redis 기반 실시간 랭킹 시스템 구현"
AI: ✅ 워크플로우 시작
    📋 문서: .claude/tasks/ACTIVE/impl-유저랭킹기능.md
```

### 2. 세션 중단 → 재개
```
[세션 1]
User: "#구현 대규모리팩토링"
AI: [Phase 1/3 완료]
[세션 종료]

[세션 2]
AI: 📋 진행 중인 작업: impl-대규모리팩토링 (33% 완료)
User: "#구현재개 대규모리팩토링"
AI: [문서 로드, Phase 2부터 계속]
```

### 3. 작업 완료
```
User: "#구현완료 유저랭킹기능"
AI: ✅ 작업 완료
    📚 COMPLETED/impl-유저랭킹기능.md로 이동
```

## 📁 파일 구조

```
.claude/tasks/
├── README.md              # 전체 가이드
├── TEMPLATE.md            # 작업 문서 템플릿
├── QUICK_START.md         # 이 파일
├── ACTIVE/                # 🟡 진행 중
├── COMPLETED/             # ✅ 완료
└── ARCHIVED/              # ❌ 폐기
```

---

## 💡 핵심 개념

1. **작업 문서 = 영구 컨텍스트**: 세션이 끊겨도 문서에서 복원
2. **Phase 기반 진행**: 단계별 체크리스트로 관리
3. **병렬 실행 지원**: 독립적인 작업은 동시 진행 가능

---

## 🎯 언제 사용하나?

### ✅ 사용 권장
- 여러 단계가 필요한 복잡한 구현
- 세션이 중단될 가능성이 있는 작업
- 나중에 참고할 필요가 있는 작업

### ❌ 사용 불필요
- 단순 버그 수정 (10분 이내)
- 단일 파일 수정

---

## 💬 FAQ

**Q: 여러 작업을 동시에 진행할 수 있나요?**
A: 네. 파일 충돌이 없으면 병렬 실행 가능합니다.

**Q: 세션이 끊기면 어떻게 되나요?**
A: 다음 세션 시작 시 자동으로 알림. `#구현재개`로 이어서 진행.
