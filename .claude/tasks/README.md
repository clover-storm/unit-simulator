# Task Workflow System

AI Agent가 복잡한 구현 작업을 추적하고 세션 간 문맥을 유지하기 위한 시스템입니다.

## 📋 사용 방법

### 작업 시작
```
User: "#구현 심볼인덱싱수정"
AI: [워크플로우 자동 생성]
    - .claude/tasks/ACTIVE/impl-심볼인덱싱수정.md 생성
    - 작업 계획 수립
    - 체크리스트 작성
```

### 병렬 작업
```
User: "#구현 유저랭킹기능"
AI: [두 번째 워크플로우 생성]
    - impl-유저랭킹기능.md 생성
    - 기존 impl-심볼인덱싱수정.md는 유지
    - 독립적으로 진행
```

### 작업 완료
```
User: "#구현완료 심볼인덱싱수정"
AI: [워크플로우 종료]
    - ACTIVE/impl-심볼인덱싱수정.md → COMPLETED/ 이동
    - 최종 상태 기록
    - 커밋 정보 저장
```

### 작업 폐기
```
User: "#구현폐기 유저랭킹기능 더 이상 필요없음"
AI: [워크플로우 폐기]
    - ACTIVE/impl-유저랭킹기능.md → ARCHIVED/ 이동
    - 폐기 사유 기록
```

---

## 🗂️ 디렉토리 구조

```
.claude/tasks/
├── README.md                    # 이 파일
├── TEMPLATE.md                  # 작업 문서 템플릿
├── ACTIVE/                      # 진행 중인 작업
├── COMPLETED/                   # 완료된 작업
└── ARCHIVED/                    # 폐기된 작업
```

---

## 🏷️ 태그 명령어

### 작업 생성
```
#구현 <작업명> [설명]
```

### 작업 재개
```
#구현재개 <작업명>
```

### 작업 완료
```
#구현완료 <작업명> [완료 노트]
```

### 작업 폐기
```
#구현폐기 <작업명> <사유>
```

### 작업 목록 조회
```
#구현목록
```

---

## 📄 워크플로우 문서 구조

각 `impl-<작업명>.md` 파일은 다음 구조를 따릅니다:

```markdown
# [IMPL] 작업명

## 📋 작업 목적
## 🎯 구현 계획
## ✅ 구현 상태
## 📝 구현 세부사항
## 💬 세션 노트
## 🔗 의존성
```

---

## 🔄 워크플로우 상태

- 🟡 **IN_PROGRESS**: ACTIVE 디렉토리, 진행 중
- ✅ **COMPLETED**: COMPLETED 디렉토리, 완료
- ❌ **ARCHIVED**: ARCHIVED 디렉토리, 폐기
